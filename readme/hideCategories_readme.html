PURPOSE & AIM

hideCategories is a Specialized contribution that allows the admin to declare categories (and all products within) as "hidden", to one degree or another.

Folders can either be hidden from the main menu (sidebox and main page) or you can go a step further and hide them from All Products, New Products, Featured Products, Advanced Search, and Best Sellers as well.

This module was designed for a very specific purpose, so its usefulness in your situation is up to you to determine. Whether or not this is of any use to your store is completely up to you to find out. hideCategories was developed by the original author with the purpose of using it for "custom packages"... ie: having custom one-off products or packages that are intended for a specific person and not the general surfing public.

Features
  • You can declare a category as:
    • Normal
      • Products display as normal
    • Hidden from menus
      • The "Hidden from menus" option means the categories side box and index listing page won't list that category for public viewing. However, the category behaves normally other than that (products within ARE listed in the "All Products" link and are available through search, etc).
    • Hidden (Direct-link only)
      • The "Hidden (Direct-link only)" option hides your hidden categories from searches and the "All", "Featured" and "New" listings as well as search results. This is similar to disabling the product/category except that it can still be seen if the customer has a direct link to the products URL.
  • Your choice is made in the normal categories admin page when you EDIT a product.
Limitations
  • Many DEFAULT Zen Cart listings, menus, and pages are included -- so "as is" this module should cover MOST store configurations. However, there may be listings, menus, or pages from other mods that the hidden status may not be applied where you might expect otherwise.
  • If there are places where you need to hide a product which is not included in this module, you can (and SHOULD) take a look at the files in this module to help guide you in how to add the hidden functionality to other lists, menus, etc.
  • Hidden products/categories are only hidden from the following sideboxes:
    • Categories
    • BestSellers
    • What's New
  • The "specials" menu link or sidebox will not be hidden. However, this should not pose any real issues with using this module since truthfully there is no reason I can think of where a shopowner would add a hidden product to their store specials.

INSTALLATION

!!!!!BACKUP YOUR STORE BEFORE YOU BEGIN!!!!!

New Installation

  1. Unzip the package to your local hard drive.
  2. Open the 1_required_install_files folder.
  3. Rename the YOUR_TEMPLATE and YOUR_ADMIN_FOLDER folders to match the name of your shop's template and admin directories.
  4. Use the included sql file to update your database (adds a new table - edits nothing).
  5. Copy all the files in the admin and includes folders to your store's respective folders. *CAUTION* Two of the admin files and one of the includes files are core file over-writes!

    The admin section doesn't offer the same flexibility that the store front end does, so this was unavoidable. This is not a big deal until you upgrade to a new version of Zen Cart and you'll have to carefully merge the edits then.

    There is only one core overwrite in the includes file. Carefully merge the content of includes/modules/pages/advanced_search_result/header.php with your existing file.

    NOTE: Many of the other files are not core files, but they may already exist if you've modified your cart already. If so, merge them as you see fit to integrate the necessary parts into your page(s).

    (copy ALL of the add-on files to your store!! Most module install issues are caused by storeowners who decide to NOT load ALL of the module files)
  6. If you followed these instructions to the letter, you are ready to use the hideCategories module. (Read the Usage tab BEFORE you use this module for the first time!)

Upgrading from 2.0 to 2.1

  • The only change from 2.0 to 2.1 is the addition of includes/modules/pages/advanced_search_result/header. IF YOU ARE ONLY UPGRADING FROM 2.0 to 2.1 the only change you have to make is to that file - there is no reason to reload all of the other files or run the sql statement again.

Upgrading from 2.0 or 2.1 to 2.2

  • There is no need to run the sql statement again, but all files should be replaced. Be sure to pay special attention to the core overwrite files.

Optional Install Files

Also included is a file which customizes the look of your hidden product product information page to help further prevent other products in your hidden categories from being "found". This custom template removes the "Next", "Previous", and "Listing" buttons/links from the product information page for your hidden products. It also removes the category images/links from the product information page. It requires using one of the other Zen Cart product types to facilitate this.

  1. Open the 2_optional_install_files folder.
  2. Rename the YOUR_TEMPLATE folder to match the name of your shop's template directory.
  3. Copy all the files in the includes folders to your store's respective folders.

To configure this new product_info template, do the following:

  1. Go to Catalog > Product Types
  2. Edit "Document - Product"
  3. Change "Product Type Name:" to "Product - Hidden"
  4. Select "Product Hidden" in the "This product type should be considered a sub-type of" dropdown

To apply the hidden product template to your hidden products follow these steps:

  1. Go to Catalog > Categories/Products
  2. Create a new category
  3. Change "Restrict to Product Type" to "Product - Hidden"
  4. Click "Add include SubCategories"
  5. Click "Save"
  6. After saving the category, set the "Category Visibility"

Un-Installation

Reverse everything you did (remove all the files remove the edits from the modified core files, restore Zen Cart defaults, etc..) Run the uninstall SQL file to remove the Hide Categories table. It is best to simply keep a good pre-install backup set so you can restore if you decide this isn't what you want.

USAGE


This contrib is very simple to use, but takes some planning to use well. From your admin panel, click the Catalog menu and choose Categories/Products. Whenever you EDIT a category, you will now be given the option to hide it (as described earlier).

FOR EXISTING STORES: You will need to open ALL your EXISTING categories and re-save them in order to apply the hideCategories changes. Otherwise some places where categories should be hidden will not work correctly, and some places where categories should NOT be hidden may not display correctly.

NOTE: The hidden feature is ONLY available when you EDIT, not when you CREATE a new category. You must create your category 1st first, then go back and edit it. You'll notice the folder icon changes to give you a visual cue that it is hidden.
